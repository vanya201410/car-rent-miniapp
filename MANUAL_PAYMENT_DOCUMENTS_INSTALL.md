# Ручная оплата и документы клиента

В этом архиве добавлено:

1. Кнопка **❓ Не могу оплатить онлайн**
   - клиент нажимает ее, если карта не проходит;
   - бронь получает статус `pending_manual_payment`;
   - админу приходит уведомление в Telegram;
   - админ может нажать **💰 Предоплата получена вручную** или отклонить бронь.

2. Блок **Документы водителя**
   - клиент загружает фото водительских прав;
   - клиент загружает фото паспорта / NIE / DNI;
   - клиент указывает телефон;
   - документы сохраняются в приватный Supabase Storage bucket `booking-documents`;
   - админу приходят временные ссылки на документы в Telegram;
   - админ может нажать **✅ Документы проверены** или **❌ Документы не подходят**.

## Что сделать после загрузки архива в GitHub

1. Открой Supabase → SQL Editor → New query.
2. Скопируй весь SQL из файла:

```text
supabase/extra-services-setup.sql
```

3. Нажми **Run**.

Это добавит новые поля в `bookings` и создаст приватный bucket `booking-documents`.

## Важно

Документы хранятся в приватном bucket. Клиент не видит прямые публичные ссылки. Админу в Telegram отправляются подписанные ссылки, которые действуют 7 дней.

## После SQL

1. Дождись Vercel deploy.
2. Открой приложение с новой версией, например:

```text
https://car-rent-miniapp.vercel.app/?v=90
```

3. Создай тестовую бронь.
4. Проверь кнопку **❓ Не могу оплатить онлайн**.
5. Проверь загрузку двух фото документов и телефона.
