# Предоплата для фиксации брони

Что добавлено:

1. В приложении показывается блок **Предоплата**.
2. После отправки заявки клиент видит сумму предоплаты и остаток при получении.
3. Новая заявка получает статус `pending_prepayment`.
4. Админу в Telegram приходит кнопка **💰 Предоплата получена**.
5. После нажатия админом:
   - `status = confirmed`
   - `prepayment_status = paid`
   - `prepayment_paid_at = now()`
   - клиент получает сообщение, что бронь подтверждена.

## Логика суммы предоплаты

- Jaguar / премиум авто / депозит от 800 € / цена от 120 €/день: **100 €**
- Обычные авто 1–3 дня: **30 €**
- Обычные авто 4+ дня: **50 €**

Предоплата входит в стоимость аренды.

Пример:

- Итого аренда: 385 €
- Предоплата: 50 €
- Остаток при получении: 335 €
- Залог оплачивается отдельно

## Что нужно сделать после загрузки в GitHub

1. Открой Supabase.
2. Перейди в **SQL Editor → New query**.
3. Открой файл `supabase/extra-services-setup.sql`.
4. Скопируй весь SQL и нажми **Run**.
5. Дождись деплоя Vercel.
6. Открой приложение с новой версией ссылки, например `?v=30`.

## Новые поля в bookings

- `prepayment_amount`
- `prepayment_status`
- `prepayment_paid_at`
- `payment_method`
- `remaining_amount`

## Новые статусы

- `pending_prepayment` — клиент отправил заявку, ждем предоплату
- `confirmed` — админ отметил, что предоплата получена
- `cancelled` — заявка отклонена или отменена
