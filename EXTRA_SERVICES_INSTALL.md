# Что добавлено

Я добавил правильную систему доп. услуг, пробег и обязательное согласие с правилами аренды:

- `api/extra-services.js` — приложение берет список услуг из Supabase `extra_services`.
- `api/create-booking.js` — сервер сам проверяет выбранные услуги, считает сумму и сохраняет строки в `booking_extra_services`.
- `src/App.jsx` — клиент видит новые доп. услуги, цены, описание, итог, включенные километры, цену доп. километра и блок правил аренды с обязательной галочкой согласия.
- `supabase/extra-services-setup.sql` — SQL для создания/обновления таблиц, услуг, полей пробега и полей согласия с правилами.

# Что сделать после загрузки в GitHub

1. Открой Supabase → SQL Editor.
2. Открой файл `supabase/extra-services-setup.sql` из проекта.
3. Скопируй весь SQL и нажми Run. Это важно: SQL добавляет поля `included_km`, `extra_km_price`, `rules_accepted`, `rules_accepted_at`, `rules_version`.
4. Загрузи проект в GitHub.
5. Дождись Vercel Deployments → Ready.
6. Открой бота и проверь бронирование.

# Как проверить

1. Выбери машину и даты.
2. Добавь `👶 Детское кресло` — к сумме должно прибавиться 25 €.
3. Добавь `👤 Дополнительный водитель` на 7 дней — должно быть 50 €, не 70 €.
4. Выбери `🛣 +100 км`, потом `🛣 +300 км` — должен остаться только один пакет.
5. Проверь блок “Пробег и правила аренды”: должно быть видно 200 км/день и 0.25 €/км сверх лимита.
6. Попробуй отправить заявку без галочки правил — кнопка должна быть неактивна.
7. Поставь галочку согласия и отправь заявку.
8. В Supabase открой `booking_extra_services` — там должны появиться выбранные услуги этой брони.
9. В Supabase открой `bookings` — там должны сохраниться `included_km`, `extra_km_price`, `rules_accepted = true`, `rules_accepted_at`, `rules_version`.

# Важно

`Расширенная страховка` убрана из списка, чтобы не создавать юридический риск. Вместо нее позже можно сделать отдельный безопасный пакет защиты с понятными условиями.
