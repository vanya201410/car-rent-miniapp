# Что добавлено

Я добавил правильную систему доп. услуг:

- `api/extra-services.js` — приложение берет список услуг из Supabase `extra_services`.
- `api/create-booking.js` — сервер сам проверяет выбранные услуги, считает сумму и сохраняет строки в `booking_extra_services`.
- `src/App.jsx` — клиент видит новые доп. услуги, цены, описание, итог и включенные километры.
- `supabase/extra-services-setup.sql` — SQL для создания/обновления таблиц и услуг.

# Что сделать после загрузки в GitHub

1. Открой Supabase → SQL Editor.
2. Открой файл `supabase/extra-services-setup.sql` из проекта.
3. Скопируй весь SQL и нажми Run.
4. Загрузи проект в GitHub.
5. Дождись Vercel Deployments → Ready.
6. Открой бота и проверь бронирование.

# Как проверить

1. Выбери машину и даты.
2. Добавь `👶 Детское кресло` — к сумме должно прибавиться 25 €.
3. Добавь `👤 Дополнительный водитель` на 7 дней — должно быть 50 €, не 70 €.
4. Выбери `🛣 +100 км`, потом `🛣 +300 км` — должен остаться только один пакет.
5. Отправь заявку.
6. В Supabase открой `booking_extra_services` — там должны появиться выбранные услуги этой брони.

# Важно

`Расширенная страховка` убрана из списка, чтобы не создавать юридический риск. Вместо нее позже можно сделать отдельный безопасный пакет защиты с понятными условиями.
